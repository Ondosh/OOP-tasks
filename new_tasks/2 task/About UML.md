# Связи внутри пакетов

Начнём с папки bot:
1. Здесь у нас имеется класс-интерфейс, которые наследуют все остальные - следовательно, связь у всех классов с ним - наследование.
2. HybridBot создаёт по одному экземпляру GigaChatBot и SimpleBot. Создаётся он сразу при создании объекта - его не вызывает метод. Следовательно, мы можем сказать, что это композиция, т.к. два класса являются частью общего класса HybridBot, и он без них существовать не может (иначе ошибка создания).

Папка Controller:
1. WelcomeController создаёт класс User, делая это в определенном методе и после передаёт его дальше. Это говорит о связи - агрегации. Метод может и не вызваться, следовательно класс не обязателен (даже если он обязателен в контексте полноценного использования программы)
2. ChatController имеет вложенный класс ProfileState

Папка model:
1. Message имеет вложенный класс enum Sender, который хранит в себе поля User, Bot.
2. User содержит в себе поле `List<Message>`, которое позже используется в методах - агрегация.

Папка util внутренних взаимодействий не имеет.

# Внешние связи

Welcome Controller:
1. User - зависимость. Создаёт в методе объект и передаёт его в ChatController.
2. SceneSwitcher - вызывает статический метод для переключения окна.

GigaChatBot:
1. UserProfile - агрегация. Объявлен в поле и используется в методе setUserProfile.

HistoryManager:
1. User - зависимость. Используется в методе Save
2. Message - зависимость. Используется во всех методах.

ChatController:
1. Message - зависимость. Используется в методе
2. User - агрегация. Объявлен в поле user и используется в других методах
3. HybridBot - агрегация. 
4. Остальное - зависимости

Main:
1. Наследует Application
2. Имеет зависимости CurrentUser, SceneSwitcher, HistoryManager, ProfileManager
