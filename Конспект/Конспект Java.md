## Что такое Java?

Java - это строго типизированный кроссплатформенный объектно-ориентированный язык программирования общего назначения. Он является одновременно интерпретируемым и компилируемым. Рассмотрим все свойства языка более подробно:
* **Строгая типизация** значит, что в языке программирования наложены ограничения на операции между переменными различающихся типов. Возьмём популярный пример с переменной string и double:
	В python "2" (string) и 2.0 (float) сложатся без проблем и результат будет переведён в float. В Java же такое сложение принципиально невозможно из-за несоответствия типов. 
		Примечание: Более того, в Python реально даже умножение строки на целое число.
* **Кроссплатформенность** - в Java работает принцип написания кода один раз и использования повсеместно. То есть, код написанный изначально для Windows, будет работать и для Linux (если, конечно, программа не запрашивает специфичных для какой-то ОС библиотек)
* **Объектно-ориентированность** языка проявляется в том, что сам язык буквально требует использовать структуру классов для создания программы. В Java буквально всё является объектом, а для создания первоначальных объектов, полей и т.п. используется особый модификатор.
* **Язык общего назначения** - значит, он не специализирован для конкретного узкого направления и может использоваться для создания любых типов ПО.
* **Интерпретируемость и компилируемость** языка выражена в том, что исходный код сначала компилируется в байт код `.class`, а после JVM интерпретирует этот байт-код на нужной платформе. Также для повышения производительности JVM анализирует часто выполняемые участки кода и динамически компилирует их в машинный код.

## Актуальный стандарт Java.

Наиболее используемыми версиями Java сейчас могут являться сразу три версии, но нужно сказать, что следующее перечисление будет коррелировать со "свежестью" версий: Java 17, Java 21, Java 25. Можно задаться вопросом, почему нельзя, например взять Java 24 или иные версии. Дело в том, что перечисленные выше версии являются LTS-версиями, то есть Long-Term Support (Долгосрочная поддержка). Такие версии получают исправления и патчи безопасности в течение нескольких лет в отличии от тех версий, которым долгосрочную поддержку обеспечить трудно.
Стоит отметить, что переход с Java 17 или 21 на новую 25 может быть трудоёмким из-за того, что старые программы не могут работать с новой версией без исправлений программистами. Переход тем сложнее, чем больше разница версий.

По сути, наше консольное приложение уже написано, разберём, что в нём происходит:
	1. Перво-наперво объявляется класс Main - здесь он по сути нужен только для объявления точки входа - метода. Далее идёт метод с модификаторами public, static и void. Разберём их:
		1. Public - говорит, что метод может использоваться чем угодно и откуда угодно. В нашей ситуации это позволяет вызывать его для того, чтобы программа смогла запуститься.
		2. Static - этот модификатор позволяет запустить метод без создания экземпляра класса - объекта.
		3. Void - говорит, что этот метод не будет возвращать какой-либо результат.
			Вывод: из первых двух мы можем понять, что мы даём разрешение Java Virtual Machine на использование метода извне, и при том без создания экземпляра класса. Без этих двух модификаторов в целом не могла бы запуститься. Также важно отметить, что название что в данном классе метод main обязательно должны называться именно так, иначе программа не поймёт как ей запускаться, т.к. первый запуск происходит именно в main.
	2. В 4 строке также в скобках есть аргумент `String[] args`. Он говорит что будет подан массив типа String, который мы будем называть `args`. Это аргументы программы, которые должны быть переданы для нормальной работы приложения (но не всегда обязательны).
	3. В самом теле программы выводится "Hello and welcome", после чего происходит 5 итераций, в каждой из которых берётся значение итерируемой переменной и выводится с приставкой "i = ".

В нашем приложении не хватает взаимодействия с аргументами, поэтому обновим его:

```Java
public class Main {  
    public static void main(String[] args) {  
        if (args.length == 1) {  
            System.out.print("Hello and welcome!");  
            for (int i = 1; i <= Integer.parseInt(args[0]); i++) {  
                System.out.println("i = " + i);  
            }  
        }  
    }  
}
```

Теперь разберём новые команды - `.length`, `Integer.parseInt`, а так же что такое `args[0]`:

- `args.length` - возвращает количество аргументов, которые 
- `Integer.parseInt` - считывает значение с переменной строкового типа (т.к. все аргументы передаются изначально именно в строковом типе)
- `args[0]` - первый переданный аргумент.

> Примечание: Да, в программе не хватает исключения ситуации ввода неправильного типа данных или ошибку в их количестве, но это будет опущено.

## Основные типы Java

Типы в Java разделяются на **примитивные** и **ссылочные**:

| Тип         | Размер            | Диапазон               | Пример                   |
| ----------- | ----------------- | ---------------------- | ------------------------ |
| **byte**    | 1 байт            | -128 до 127            | `byte b = 100;`          |
| **short**   | 2 байта           | -32,768 до 32,767      | `short s = 30000;`       |
| **int**     | 4 байта           | -2³¹ до 2³¹-1          | `int i = 1000000;`       |
| **long**    | 8 байт            | -2⁶³ до 2⁶³-1          | `long l = 10000000000L;` |
| **float**   | 4 байта           | ±3.4E-38 до ±3.4E+38   | `float f = 3.14f;`       |
| **double**  | 8 байт            | ±1.7E-308 до ±1.7E+308 | `double d = 3.14159;`    |
| **char**    | 2 байта           | 0 до 65,535 (Unicode)  | `char c = 'A';`          |
| **boolean** | 1 бит (логически) | true / false           | `boolean flag = true;`   |
	Небольшое примечание: float пишется обязательно с f на конце, т.к. по умолчанию все дробные являются double, а long из-за int пишется с L на конце..

| Категория               | Примеры                                     |
| ----------------------- | ------------------------------------------- |
| **Классы**              | `String`, `Integer`, `ArrayList`, `Scanner` |
| **Интерфейсы**          | `List`, `Map`, `Runnable`, `Comparable`     |
| **Массивы**             | `int[]`, `String[]`, `double[][]`           |
| **Перечисления (enum)** | `enum Day { MON, TUE, WED }`                |

Первая таблица представляет собой список примитивных, а вторая - ссылочных. Ссылочные типы хранят ссылку на объект в памяти, а примитивы - само значение. Язык Java со статической типизацией, поэтому во время выполнения программа не может просто так поменять тип переменной, он должен быть известен на этапе компиляции. Также он строго типизированный, что значит, что преобразования типов происходят там, где они безопасны. Нельзя, например, перевести очень большое число из long в int, т.к. произойдёт потеря данных.

## Коллекции стандартной библиотеки

**Iterable** (интерфейс)
- **Collection** (интерфейс)
    - **List** (упорядоченная, с дубликатами, по индексу)
        - ArrayList
        - LinkedList
    - **Set** (уникальные, без порядка)
        - HashSet
        - LinkedHashSet (с порядком добавления) 
        - TreeSet (отсортированный)
    - **Queue** (очередь)
        - PriorityQueue (по приоритету)
        - ArrayDeque (двусторонняя очередь)
        - LinkedList (также реализует Queue) 
- **Map** (пары ключ-значение) - не наследует Collection
    - HashMap
    - LinkedHashMap (с порядком добавления)
    - TreeMap (отсортированный по ключам)

Сначала стоит уточнить что такое `Collection` в этой структуре - это корневой интерфейс для групп объектов, но поскольку `Map` не хранит в себе группы, а пары "ключ-значение", они туда не входят.

Разберём все коллекции по порядку:

#### Списки

`ArrayList` - массив с динамическим размером.

```Java
// Создание
List<String> list = new ArrayList<>();

// Добавление
list.add("Apple");           // в конец
list.add(0, "Banana");       // по индексу
list.addAll(Arrays.asList("Cherry", "Date"));

// Получение
String fruit = list.get(0);  // "Banana"
int size = list.size();      // 4

// Удаление
list.remove(0);              // по индексу
list.remove("Apple");        // по значению

// Проверка
boolean hasApple = list.contains("Apple");
int index = list.indexOf("Cherry");

// Итерация
for (String item : list) {
    System.out.println(item);
}

// Сортировка
Collections.sort(list);
list.sort(String::compareTo);
```

>Примечание: когда мы создаём ссылочный объект, мы можем не писать дважды тип создаваемого объекта. То есть, когда мы в левой части написали тип в `<>` вписали тип, в правой можно его не дублировать и оставить пустым - он "впишется" самостоятельно.

`Linked List` - двусвязный список. У него те же методы, что и у ArrayList, но также добавлены и другие.

```Java
List<Integer> list = new LinkedList<>();

list.add(1);
list.add(2);

list.addFirst(0);   // специфично для LinkedList
list.addLast(3);
int first = list.getFirst();
int last = list.getLast();
```

#### Множества

`HashSet` - коллекция, которая не может содержать дубликаты

```Java
import java.util.*;

public class HashSetDemo {
    public static void main(String[] args) {
        // Создание
        Set<String> set = new HashSet<>();
        
        // Добавление
        set.add("Apple");
        set.add("Banana");
        set.add("Orange");
        set.add("Apple");  // Игнорируется, дубликат
        
        System.out.println(set);  // порядок произвольный
        
        // Проверка
        System.out.println(set.contains("Banana"));  // true
        System.out.println(set.size());              // 3
        
        // Удаление
        set.remove("Orange");
        
        // Итерация
        for (String fruit : set) {
            System.out.println(fruit);
        }
        
        // Очистка
        set.clear();
        System.out.println(set.isEmpty());  // true
    }
}
```

`LinkedHashSet` - упорядоченная коллекция без дубликатов.

```Java
public class LinkedHashSetDemo {
    public static void main(String[] args) {
        Set<String> linkedSet = new LinkedHashSet<>();
```

#### Очереди

**Queue** (Очередь) - это коллекция, предназначенная для хранения элементов в определённом порядке.

#### Карты

**Map** — это коллекция, хранящая пары "ключ -> значение". Ключи уникальны, каждому ключу соответствует одно значение.

HashMap - Основан на хэш-таблице. Порядок НЕ гарантирован

```Java
public class HashMapDemo {
    public static void main(String[] args) {
        // Создание
        Map<String, Integer> map = new HashMap<>();
        
        // Добавление
        map.put("One", 1);
        map.put("Two", 2);
        map.put("Three", 3);
        map.put("One", 10);  // Перезаписывает значение
        
        System.out.println(map);  // {One=10, Two=2, Three=3} (порядок произвольный!)
        
        // null ключ и null значение
        map.put(null, 0);
        map.put("Four", null);
        System.out.println(map);  // {null=0, One=10, Two=2, Three=3, Four=null}
        
        // Получение
        System.out.println(map.get("Two"));      // 2
        System.out.println(map.get("Five"));     // null
        
        // Удаление
        map.remove("Three");
        
        // Итерация по ключам
        for (String key : map.keySet()) {
            System.out.println(key + " = " + map.get(key));
        }
        
        // Итерация по значениям
        for (Integer value : map.values()) {
            System.out.println(value);
        }
        
        // Итерация по записям (эффективно)
        for (Map.Entry<String, Integer> entry : map.entrySet()) {
            System.out.println(entry.getKey() + " -> " + entry.getValue());
        }
    }
}
```

## Классы

Класс - это шаблон для создания объектов. Он объединяет в себе поля и методы (данные и поведение) в одной структуре. Класс является ссылочным типом, т.к. указательных типов в Java не существует.

Класс - это тип данных, определяемый пользователем, а объект - это экземпляр.

Пример класса (с описанием):

```Java
/* Класс представляющий информацию о каком-либо объекте и его количестве.
 *
 *  @author - User
 */
class Thing {
    String thing;      // поле - название предмета, thing - это ссылка на объект, который ещё не создан. Вернее - на null.
    int amount;        // поле - количество. int - примитивный тип. По умолчанию = 0
    
    void start() {     // метод - действие предмета
        System.out.println("It exists.");
    }
}
```

#### Особенности классов в Java

1. Только один публичный класс в файле (имя класса совпадает с именем файла).
2. Конструктор вызывается автоматически при создании объекта.
3. Перегрузка методов.
	В одном классе можно создать несколько методов с одинаковым именем, но разными параметрами.
4. Переопределение методов.
	Дочерний класс может изменить реализацию метода родителя.
5. Ключевое слово this.
	Ссылается на текущий объект внутри класса.
6. Статические поля и методы (static) 
	Принадлежат классу, а не объекту. Существуют в единственном экземпляре для всех объектов класса. Статические методы могут обращаться только к статическим полям и методам. Обращение через имя класса, а не через объект.
7. Модификаторы доступа  
	Private — только внутри класса. Default (без модификатора) — внутри пакета. Protected — внутри пакета + наследники. Public — везде.
8. Наследование (extends)  
	Класс может наследовать только один родительский класс (одиночное наследование). Все классы неявно наследуются от Object. Конструкторы не наследуются, их нужно вызывать через super()
9. Абстрактные классы  
	Нельзя создать экземпляр абстрактного класса. Может содержать абстрактные методы (без реализации) и обычные методы. Предназначен для того, чтобы от него наследовались другие классы.
10. Final классы и методы  
	Final класс нельзя наследовать. Final метод нельзя переопределить. Final поле нельзя изменить после инициализации (константа).
11. Порядок инициализации  
	Сначала статические поля и статические блоки (в порядке объявления), потом нестатические поля и блоки инициализации, потом конструктор.
12. Вложенные классы  
	Бывают статические (не требуют внешнего объекта) и внутренние (требуют объект внешнего класса). Анонимные классы создаются на месте без имени. Локальные классы объявляются внутри метода.
13. Классы-обёртки  
	Для каждого примитива есть класс-обёртка (int → Integer, double → Double). Автоупаковка автоматически преобразует примитив в объект и обратно.
14. Record классы (Java 14+)  
	Неизменяемые классы-носители данных. Автоматически создают конструктор, геттеры, equals, hashCode, toString.
15. Enum классы  
	Ограниченное количество предопределённых констант. Могут иметь поля, методы и конструкторы. Нельзя создать экземпляр enum через new.
16. Объекты создаются в куче (heap)  
	Ссылки на объекты хранятся в стеке (stack), сами объекты в куче. При отсутствии ссылок объект удаляется сборщиком мусора.
17. Классы загружаются лениво  
	Класс загружается в память при первом использовании (создание объекта, вызов статического метода, обращение к статическому полю).
18. Метод main — точка входа  
	Должен быть public static void main(String[] args). Только такой сигнатуры Java распознаёт как точку входа.
19. Классы не могут быть private или protected  
	Только public или без модификатора (package-private). Исключение — вложенные классы, они могут иметь любые модификаторы.

## Модульное тестирование в Java

Для того, чтобы начать модульное тестирование, сначала необходимо скачать библиотеку для него. Поскольку мы работаем в Maven, пример добавления библиотеки будет показан именно на этой системе сборки.

1. Открываем гамбургер-меню в верхней левой части приложения, нажимаем `File -> Project Structure`. Должно появиться следующее окно:

![[Pasted image 20260423093233.png]]

2. Нажимаем галочку напротив `org.junit,jupiter...5.12.1` и после жмём Apply.
3. Поскольку некоторых нужных зависимостей нет, придётся напрямую воспользоваться возможностью редактирования `pom.xml` и добавить следующий код:

```xml
<dependency>  
    <groupId>org.junit.jupiter</groupId>  
    <artifactId>junit-jupiter-params</artifactId>  
    <version>5.10.0</version>  <!-- Убедитесь, что версия совпадает с вашими другими JUnit-зависимостями -->  
    <scope>test</scope>  
</dependency>
```

4. После нажимаем на меню Maven и выбираем `Sync All Maven Projects`.

![[Pasted image 20260423094206.png]]

Наконец, мы можем написать тесты для нашего приложения. Возьмём простой класс `Calculator.java`:

```Java
package com.example;  
  
public class Calculator {  
  
    public int add(int a, int b) {  
        return a + b;  
    }  
  
    public int subtract(int a, int b) {  
        return a - b;  
    }  
  
    public int multiply(int a, int b) {  
        return a * b;  
    }  
  
    public int divide(int a, int b) {  
        if (b == 0) {  
            throw new IllegalArgumentException("Cannot divide by zero");  
        }  
        return a / b;  
    }  
}
```

Необходимо его протестировать. Создадим в папке рядом с ним файл `CalculatorTest.java`.

```Java
package com.example;  
  
import org.junit.jupiter.api.*;  
import org.junit.jupiter.params.ParameterizedTest;  
import org.junit.jupiter.params.provider.CsvSource;  
import static org.junit.jupiter.api.Assertions.*;  
  
class CalculatorTest {  
  
    // Параметризованный тест с несколькими наборами данных  
    @ParameterizedTest  
    @CsvSource({  
            "1, 1, 2",  
            "0, 5, 5",  
            "-3, 7, 4",  
            "100, -50, 50"  
    })  
    @DisplayName("Параметризованный тест сложения")  
    void testAddWithMultipleInputs(int a, int b, int expected) {  
        Calculator calc = new Calculator();  
        assertEquals(expected, calc.add(a, b));  
    }  
  
    // Группировка связанных тестов  
    @Nested  
    @DisplayName("Тесты деления")  
    class DivideTests {  
  
        @Test  
        @DisplayName("Деление положительных чисел")  
        void dividePositiveNumbers() {  
            Calculator calc = new Calculator();  
            assertEquals(4, calc.divide(20, 5));  
        }  
  
        @Test  
        @DisplayName("Деление отрицательных чисел")  
        void divideNegativeNumbers() {  
            Calculator calc = new Calculator();  
            assertEquals(-4, calc.divide(-20, 5));  
        }  
    }  
}
```

Рассмотрим новые аннотации для тестирования кода:
- @ParameterizedTest - говорит, что следующий тест будет проходить с несколькими наборами данных
- @CsvSource - даёт список из нескольких наборов значений, где сами наборы находятся в кавычках, где внутри значения разделяются запятой, сами наборы разделяются запятой тоже, предварительно закрывая набор кавычкой.
- @Nested - позволяет группировать несколько тестов. Нужен для того, чтобы организовать большое количество тестов.
- @DisplayName - Даёт имя этой группе тестов.
- @Test - самая часто используемая аннотация, говорящая о том, что следующий метод является тестовым.

Также есть иные аннотации, которые не были приведены, но рекомендуются к рассмотрению:
- @BeforeAll
- @BeforeEach
- @AfterEach
- @AfterAll
- @RepeatedTest
- @TestFactory
- @Disabled

Другие функции для тестирования:

1. `assertEquals` - единственная рассмотренная в коде функция, первым аргументом запрашивает ожидаемый результат, вторым функцию для использования (с нужными аргументами). Проверяет равен ли тест ожидаемому числу.
2. `assertNotEquals` - тоже самое что 1, но ожидает что не равен.
3. `assertThrows` - запрашивает минимум 2 аргумента - класс ожидаемого исключения и код, который должен его бросить, 3 аргумент - сообщение об ошибке при неудачном тесте, 4 - "ленивое" сообщение (при падении теста). Ожидает, что код выбросит исключение.
4. `assertDoesNotThrow` - ожидает, что код не выбросит исключение
5. `assertSame` - ожидает, что два объекта представляют из себя одно и то же.
6. `assertNotSame` - ожидает, что будут поданы два разных объекта.
7. `assertTrue` - ожидает, что условие истинно.
8. `assertFalse` - ожидает, что условие ложно.
9. `assertNull` - ожидает, что объект равен `Null`.
10. `assertNotNull` - ожидает, что объект не равен `Null`.
11. `assertArrayEquals` - сравнивает массивы, ожидает что они будут одинаковы.
12. `assertAll` - выполняет несколько проверок, даже если некоторые падают.
13. `assertTimeout` - проверка, что код выполнился быстрее указанного времени.
14. `assertTimeoutPreemptively` - прерывает выполнение кода при превышении ожидаемого времени.
15. `assertIterableEquals` - проверка итерируемых объектов (например List)
16. `assertLinesMatch` - сравнение строк (включая regex)
17. `assertInstanceOf` - проверка является ли объект экземпляром определённого класса.
18. `assertNotInstanceOf` - проверка что объект не является экземпляром конкретного класса.

## Road-maps для изучения Java

Дорожная карта сама по себе это стратегический документ, который описывает основные цели, этапы для реализации продукта или какого-либо конечного результата. 
В нашем случае дорожная карта нужна для того, чтобы расставить точки над "и", то есть понять чёткий план изучения всех аспектов языка, фреймворков и т.п.
